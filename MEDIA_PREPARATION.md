# Подготовка медиа файлов для GitHub

## 📁 Структура папок

Создайте следующую структуру в папке `public`:

```
public/
├── images/
│   ├── mountain-poster.jpg
│   ├── mountain-view-1.jpg
│   ├── mountain-view-2.jpg
│   ├── mountain-view-3.jpg
│   ├── mountain-view-4.jpg
│   ├── mountain-view-5.jpg
│   ├── mountain-view-6.jpg
│   └── mountain-view-7.jpg
└── videos/
    └── mountain-hero.mp4
```

## 🎥 Подготовка видео

### Требования для GitHub:
- **Максимальный размер**: 100 MB
- **Формат**: MP4
- **Кодек**: H.264
- **Длительность**: Рекомендуется 10-30 секунд

### Как сжать видео:

#### 1. Используя FFmpeg (рекомендуется):
```bash
# Сжать видео до 50MB
ffmpeg -i input.mp4 -c:v libx264 -crf 28 -c:a aac -b:a 128k -movflags +faststart mountain-hero.mp4

# Сжать до 30MB
ffmpeg -i input.mp4 -c:v libx264 -crf 32 -c:a aac -b:a 96k -movflags +faststart mountain-hero.mp4
```

#### 2. Используя онлайн сервисы:
- [CloudConvert](https://cloudconvert.com/mp4-converter)
- [FreeConvert](https://www.freeconvert.com/mp4-converter)
- [Online-Convert](https://www.online-convert.com/)

#### 3. Используя VLC:
1. Откройте VLC
2. Медиа → Конвертировать/Сохранить
3. Добавьте файл
4. Нажмите "Конвертировать/Сохранить"
5. Выберите профиль "Video - H.264 + MP3 (MP4)"
6. Настройте качество для уменьшения размера

## 🖼️ Подготовка изображений

### Требования:
- **Формат**: JPG или WebP
- **Размер**: 1200x800px (рекомендуется)
- **Качество**: 80-90%
- **Размер файла**: < 2MB каждое

### Как оптимизировать изображения:

#### 1. Используя онлайн сервисы:
- [TinyPNG](https://tinypng.com/) - сжатие без потери качества
- [Squoosh](https://squoosh.app/) - Google's image optimizer
- [Compressor.io](https://compressor.io/)

#### 2. Используя Photoshop:
1. Файл → Экспортировать → Экспортировать как
2. Выберите JPG
3. Установите качество 80-90%
4. Установите размер 1200x800px

#### 3. Используя GIMP (бесплатно):
1. Файл → Экспортировать как
2. Выберите JPG
3. Установите качество 85
4. Измените размер: Изображение → Размер изображения

## 🚀 После подготовки

1. **Поместите файлы** в соответствующие папки
2. **Проверьте размеры** файлов
3. **Запустите проект**: `npm run dev`
4. **Проверьте загрузку** всех медиа файлов
5. **Зафиксируйте изменения**: `git add . && git commit -m "Add media files"`
6. **Отправьте в GitHub**: `git push origin master`

## ⚠️ Важные замечания

- GitHub имеет лимит 100MB на файл
- Если файл больше 50MB, GitHub покажет предупреждение
- Рекомендуется использовать Git LFS для больших файлов
- Всегда проверяйте размеры перед коммитом

## 🔧 Альтернатива: Git LFS

Если файлы все еще слишком большие:

```bash
# Установите Git LFS
git lfs install

# Отслеживайте большие файлы
git lfs track "*.mp4"
git lfs track "*.jpg"

# Добавьте .gitattributes
echo "*.mp4 filter=lfs diff=lfs merge=lfs -text" >> .gitattributes
echo "*.jpg filter=lfs diff=lfs merge=lfs -text" >> .gitattributes
```
